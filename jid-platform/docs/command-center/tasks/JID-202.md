# JID-202 — Honest Surface Closure

## Objective

Remove or hide visible fabricated and unfinished Lammah, Career Canvas, CV, and public-search states so every remaining control and value is truthful, localized, and supported by real behavior.

## Business reason

Friends-and-family users must not see fake recommendations, metrics, activities, or controls that imply unavailable actions.

## Constitutional sources

- Constitution Articles 3, 4, 6, 7, 9, 10, and 11.
- Feature Ledger `IND-05`, `LAM-01`, `CV-01`, `SEARCH-01`, and `I18N-01` in [FEATURE_LEDGER.md](../FEATURE_LEDGER.md).
- [TASK_BOARD.md](../TASK_BOARD.md).

## Prerequisites

- JID-101 quality gate is complete.
- Inventory every visible placeholder, fake item/metric, dead control, source query, shared empty state, and AR/EN key in the owned surfaces.

## Dependency gates

- JID-101: `COMPLETE`.
- No database gate is required for visibility-only subpackages that neither read nor write a database.

## Scope

- Hide Lammah entry points or use honest noninteractive unavailable/empty states when no traceable records exist.
- Hide or disable unfinished Career Canvas and CV controls without creating a parallel Career Record source.
- Remove hardcoded public-search copy and cover placeholder/loading/empty/error/instructions/accessibility labels in both locales.
- Reuse shared shell/empty states and add focused visibility/interaction/localization tests.

## Out of scope

- Lammah crawler/recommendations/provider integration, a new data model, Career Canvas as a separate product/feed, CV generation redesign, search ranking/data-source changes, Supabase, or deployment.

## Environment

- Isolated approved non-production branches/worktrees; local application tests only.
- No Supabase, databases, migrations, Vercel, production, `main`, or secrets.

## Permissions

- Surface-specific components/routes/tests, unavoidable namespace-specific AR/EN keys, and reports only.
- Split subpackages may own non-overlapping surfaces but must preserve this packet's acceptance criteria.

## Expected file domains

- Lammah components/routes/tests; Career Canvas/CV components/routes/tests; realtime-search components/tests; unavoidable namespace locale keys; `docs/command-center/reports/JID-202_REPORT.md` or approved subpackage reports.

## Tests

- Assert fabricated Lammah records/metrics never render.
- Assert unfinished Career Canvas/CV actions are hidden or truly noninteractive and canonical facts are not duplicated.
- Assert AR/EN search states and accessible labels with unchanged routing/keyboard behavior.
- Run focused tests, `git diff --check`, type-check, lint, and build.

## Acceptance criteria

- No visible fabricated item, company, opportunity, count, score, recommendation, percentage, status, or activity remains.
- No control promises unavailable save/generate behavior.
- All remaining dynamic content has a traceable source; absent sources yield honest noninteractive states or no entry point.
- Both locales and required gates pass.

## Evidence

- Report each removed/hidden element, its prior data source or absence, reused shared components, changed files, interaction/localization tests, and exact command results.

## Stop conditions

- Stop if honest behavior requires inventing data, crawler/provider work, a parallel canonical model, database access, out-of-domain shared edits, or two repairs fail for the same cause.

Codex on approved JID-202 surface subpackage branches.

## Verifier
